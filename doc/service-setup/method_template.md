# 서비스 셋업 가이드 — 제작 방법론(표준 템플릿)

> 목적: 외부 서비스(예: Cloudflare, Supabase, GitHub, Google Cloud)의 **회원가입 → 기본설정**을
> 사용자가 보면서 따라 할 수 있도록 **HTML 매뉴얼 + 마우스포인터 애니메이션 영상**으로 제작하는 표준 절차.
> 1편(Cloudflare) 제작 경험을 바탕으로 정리. 다른 서비스에도 그대로 적용한다.

---

## 1. 산출물 형식 (서비스 1개당)

| 산출물 | 파일 | 설명 |
|---|---|---|
| HTML 매뉴얼 | `<service>_setup_manual.html` | 단계별 화면 + 마우스포인터 + 상세 설명. 자체 완결형(이미지·영상 내장) |
| 워크스루 영상 | `<service>_setup_walkthrough.mp4` | 마우스 이동·클릭·하이라이트 애니메이션 + 자막. 약 30~45초 |
| (공유) 방법론 | `method_template.md` | 본 문서 |

화면 구성: 상단 브라우저 크롬(주소창) + 본문 + 하단 자막바(STEP n/N, 제목, 설명). 1280×972px.

---

## 2. 캡처 소스 결정 (중요)

서비스마다 라이브 캡처 가능 여부가 다르다. 시작 전 아래로 분기한다.

1. **로그인 대시보드가 자동화 브라우저에서 잘 뜨는 경우** (GitHub, Google Cloud 등 다수)
   → **실제 스크린샷**을 캡처해 사용. 단, 개인정보(이메일·계정ID·실데이터)는 가린다.
2. **가입 페이지가 봇 차단(Turnstile/CAPTCHA)·SPA 로딩으로 캡처 불가** (Cloudflare 가입 폼이 그랬음)
   → 해당 단계는 **공식 문서 기준 재현(reconstruction)**. 봇 차단·CAPTCHA는 절대 우회하지 않는다.
3. **이미 로그인되어 가입 화면 리다이렉트** → 가입 파트는 로그아웃 후 캡처하거나 재현으로 대체.

> 원칙: 정확도는 **공식 문서(developers.\<service\>.com)**에서 가져오고, 시각화는 재현/실캡처로 채운다.
> 재현 화면은 매뉴얼에 "재현(representative)"임을 반드시 명시한다.

---

## 3. 제작 파이프라인 (재사용 코드)

`outputs/`의 다음 스크립트가 엔진이다. 서비스별로 `*_screens.py`만 새로 작성하면 된다.

- **`cf_lib.py`** — 공통 렌더 엔진. 브라우저 크롬, 마우스 커서(화살표), 하이라이트 링(글로우+펄스),
  자막바, 버튼/입력필드/카드/로고 헬퍼. 브랜드 색만 바꾸면 타 서비스 재사용.
- **`cf_screens.py`** — 서비스별 화면 정의. 각 단계 = `{id, url, draw(), target(x,y,w,h), title, body}`.
  `target`이 마우스가 도착할 클릭 위치 = 하이라이트 위치.
- **`render.py`** — 단계별 정지 프레임(주석 포함) PNG/SVG 생성 → HTML 매뉴얼용.
- **`video.py`** — 타이틀 카드 → (단계마다: 커서 이동 → 하이라이트 등장 → 클릭 펄스×2 → 자막 정지) → 엔딩 카드.
  프레임을 cairosvg로 렌더 후 ffmpeg로 mp4 인코딩(20fps, H.264, yuv420p).
- **`build_html.py`** — PNG·mp4를 base64로 내장한 단일 HTML 매뉴얼 생성.

필요 패키지: `cairosvg`, `Pillow`, `ffmpeg`, 한글 폰트(Noto Sans CJK KR).

### 새 서비스 추가 절차
1. 공식 문서에서 가입·기본설정 단계 수집(WebSearch/web_fetch).
2. 캡처 소스 결정(2장) — 실캡처 가능분은 Chrome으로 캡처, 불가분은 재현.
3. `<service>_screens.py`에 단계별 화면·타깃·자막 정의(브랜드 색 교체).
4. `render.py`·`video.py`·`build_html.py` 실행 → 검수 → 작업 폴더로 복사.

## 4. 품질 체크리스트
- [ ] 각 단계의 클릭 대상이 마우스포인터·하이라이트로 명확히 지시되는가
- [ ] 자막/설명이 공식 문서와 일치하는가(버튼 영문 라벨 그대로 + 한글 설명)
- [ ] 개인정보·실계정 데이터가 노출되지 않는가
- [ ] 재현 화면은 "재현"임을 표기했는가
- [ ] 영상이 모든 단계와 끝맺음(완료 카드)을 포함하는가
- [ ] HTML이 단독 파일로 열려도 이미지·영상이 보이는가

---

## 5. 안전 원칙
- 계정 생성·비밀번호 입력·결제·권한 변경 등은 **사용자가 직접** 수행(에이전트 대행 금지).
- 봇 차단·CAPTCHA 우회 금지.
- 로그인 화면 캡처가 필요하면 사용자가 로그인한 뒤 진행.

# 16. 초보자용 전자책 기획 — 「바이브코딩으로 시작하는 1인 SaaS」

> 기존 `products.ebook-vibe` / `resources.ebook` (상태: 준비중)의 **1차 완성본(5페이지 요약본)**.
> 산출물: `public/ebook/vibe-1in-saas.html` — 화면에서 바로 읽고, 인쇄(Ctrl+P)하면 A4 5쪽 PDF로 저장된다.

---

## 1. 기획 의도

| 항목 | 내용 |
|---|---|
| 독자 | 코드를 한 줄도 안 써 본 비개발자. "AI로 뭔가 만들 수 있다"는 말만 들어본 상태 |
| 약속 | 다 읽고 나면 **내 아이디어 한 문장 + 첫 프롬프트 + 배포까지의 지도**를 손에 쥔다 |
| 분량 | A4 5쪽. 앉은 자리에서 15분이면 끝 (길면 안 읽는다) |
| 톤 | 존댓말·비유 중심. 전문 용어는 반드시 괄호로 풀어쓴다 |
| 가격 | 무료(교육 목적). 후원은 선택 |

**설계 원칙 3가지**
1. 한 페이지 = 한 가지 결론. 페이지마다 "이 장에서 얻는 것"을 상단에 명시한다.
2. 읽고 끝나지 않게, 각 페이지에 **직접 채우는 칸(빈칸 템플릿)** 을 하나씩 둔다.
3. 초보자가 막히는 지점(용어·프롬프트·환경변수·배포)을 각각 한 페이지씩 정면으로 다룬다.

## 2. 페이지 구성 (5쪽)

| 쪽 | 제목 | 핵심 결론 | 포함 자료 |
|---|---|---|---|
| 1 | 표지 · 이 책 사용법 | "코드가 아니라 문장을 쓰는 시대다" | 독자 체크리스트, 5쪽 지도, 준비물 3가지(15분 세팅) |
| 2 | 1장 — 바이브코딩이 뭔가요 | AI에게 **무엇을 왜** 만들지 말로 설명하면, 코드는 AI가 쓴다 | 초보자 용어사전 8개, 도구 지도(에디터·DB·배포), 3원칙 |
| 3 | 2장 — 아이디어를 한 문장으로 | 만들기 전에 정리하지 않으면 AI도 못 만든다 | 한 문장 기획 템플릿, 기능 3개로 줄이기, 나쁜 예/좋은 예 |
| 4 | 3장 — AI에게 제대로 시키는 법 | 프롬프트는 4칸(맥락·목표·제약·결과형식)으로 쓴다 | 프롬프트 4단 공식, 복붙용 예시 3종, 막힐 때 대처 4가지 |
| 5 | 4장 — 세상에 내보내기 | 내 컴퓨터에서 도는 건 아직 서비스가 아니다 | 배포 3단계, 환경변수 개념, 출시 전 체크리스트 10개, 다음 단계 |

친근함을 높이기 위해 `public/characters/HT-###.png`(→ `doc/17_character_platform.md`)를 본문에 넣었다.
경로는 `../characters/HT-###.png` **상대경로** — 정적 파일이라 `/c/HT-###` 라우트에 의존하지 않고,
로컬에서 파일을 직접 열어도(file://) 깨지지 않는다.

**전원 오터로 통일한다.** 처음엔 1장 역할 카드에 메이커(HT-032·HT-033)를 섞었는데, 한 쪽 안에서
캐릭터 종이 바뀌면 시선이 흔들린다. 지금은 표지부터 맺음말까지 모두 오터 + 소품이다.

### 배치 규칙 — 잘리지 않게, 그러나 쪽 높이는 그대로

캐릭터가 **오른쪽에서 잘려 보이던 원인은 두 가지**였고 둘 다 원인 쪽에서 고쳤다.

1. 원본 컷 자체가 시트에서 꼬리·귀가 잘린 채 떨어져 있었다 → 추출 파이프라인에서 보정
   (`doc/17_character_platform.md` 5-1). 발밑 드롭섀도가 남아 표지 그라데이션 위에서 뿌옇게
   비치던 것도 같이 해결됐다.
2. 장 도입부 아바타를 `position:absolute`로 제목 줄에 붙여 두어, 이미지가 쪽 위쪽 패딩 밖으로
   삐져나와 머리가 잘렸다 → **캐릭터는 언제나 자기 칸(`.av-slot`)을 갖는다**로 바꿨다.

`.av-slot`은 크기가 먼저 정해진 flex 칸이고, 그 안에 이미지를 밑변 기준으로 세운다. 뒤에는 옅은
원형 후광을 깔아 색 있는 배경 위에서도 윤곽이 또렷하다. 칸이 텍스트 블록보다 낮게 잡혀 있으므로
**쪽 높이는 늘어나지 않는다**.

- 장 도입부 → `.ch-head` = 제목 블록 + 84px 칸. 제목·리드가 이미 그만큼 높아 높이 기여 0
- 박스(goal·trap) → 34px 칸. 본문 2줄(≈36px)보다 낮게
- 카드 → `float:right`. 제목·본문이 캐릭터를 피해 흐르므로 글자가 겹치지 않는다
- **인쇄 시 `.page`는 `height:296.6mm; overflow:hidden`** — 1px만 넘쳐도 잘린다

검증(변경 후 반드시 실행):

```bash
# ① 인쇄 → 정확히 5쪽인지
"/c/Program Files/Google/Chrome/Application/chrome.exe" --headless=new --disable-gpu \
  --no-proxy-server --no-pdf-header-footer --print-to-pdf=out.pdf \
  --virtual-time-budget=6000 file:///.../public/ebook/vibe-1in-saas.html

# ② 쪽 높이 → 폭 880px로 통짜 캡처한 뒤, 쪽 바깥 여백(x=45)에서 배경색이 끊기는 구간을 재면
#    각 쪽이 1123px(297mm) + 그림자 15px = 1138px 이어야 한다. 넘으면 그만큼 잘린다는 뜻.
```

# 16. 초보자용 전자책 기획 — 「바이브코딩으로 시작하는 1인 SaaS」

> 기존 `products.ebook-vibe` / `resources.ebook` (상태: 준비중)의 **1차 완성본(5페이지 요약본)**.
> 산출물: `public/ebook/vibe-1in-saas.html` — 화면에서 바로 읽고, 인쇄(Ctrl+P)하면 A4 5쪽 PDF로 저장된다.

---

## 1. 기획 의도

| 항목 | 내용 |
|---|---|
| 독자 | 코드를 한 줄도 안 써 본 비개발자. "AI로 뭔가 만들 수 있다"는 말만 들어본 상태 |
| 약속 | 다 읽고 나면 **내 아이디어 한 문장 + 첫 프롬프트 + 배포까지의 지도**를 손에 쥔다 |
| 분량 | A4 5쪽. 앉은 자리에서 15분이면 끝 (길면 안 읽는다) |
| 톤 | 존댓말·비유 중심. 전문 용어는 반드시 괄호로 풀어쓴다 |
| 가격 | 무료(교육 목적). 후원은 선택 |

**설계 원칙 3가지**
1. 한 페이지 = 한 가지 결론. 페이지마다 "이 장에서 얻는 것"을 상단에 명시한다.
2. 읽고 끝나지 않게, 각 페이지에 **직접 채우는 칸(빈칸 템플릿)** 을 하나씩 둔다.
3. 초보자가 막히는 지점(용어·프롬프트·환경변수·배포)을 각각 한 페이지씩 정면으로 다룬다.

---

## 2. 페이지 구성 (5쪽)

| 쪽 | 제목 | 핵심 결론 | 포함 자료 |
|---|---|---|---|
| 1 | 표지 · 이 책 사용법 | "코드가 아니라 문장을 쓰는 시대다" | 독자 체크리스트, 5쪽 지도, 준비물 3가지(15분 세팅) |
| 2 | 1장 — 바이브코딩이 뭔가요 | AI에게 **무엇을 왜** 만들지 말로 설명하면, 코드는 AI가 쓴다 | 초보자 용어사전 8개, 도구 지도(에디터·DB·배포), 3원칙 |
| 3 | 2장 — 아이디어를 한 문장으로 | 만들기 전에 정리하지 않으면 AI도 못 만든다 | 한 문장 기획 템플릿, 기능 3개로 줄이기, 나쁜 예/좋은 예 |
| 4 | 3장 — AI에게 제대로 시키는 법 | 프롬프트는 4칸(맥락·목표·제약·결과형식)으로 쓴다 | 프롬프트 4단 공식, 복붙용 예시 3종, 막힐 때 대처 4가지 |
| 5 | 4장 — 세상에 내보내기 | 내 컴퓨터에서 도는 건 아직 서비스가 아니다 | 배포 3단계, 환경변수 개념, 출시 전 체크리스트 10개, 다음 단계 |

## 3. 반복 UI 요소

- **이 장에서 얻는 것** — 페이지 상단 초록 박스
- **✍️ 직접 채워보세요** — 점선 입력 칸 (인쇄해서 손으로 쓰는 용도)
- **💡 초보자 함정** — 노란 경고 박스, 페이지당 1개
- 하단 러닝 푸터 — `habitree · vibe.habitree.io` + 쪽 번호

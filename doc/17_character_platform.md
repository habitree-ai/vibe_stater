# 17. 캐릭터 자산 플랫폼 — 채번(HT-###) 기반 이미지 관리

> 원본 가이드 시트 2장(`img/man1.png`, `img/sd_1.png`)의 캐릭터를 **개별 고화질 컷으로 분리**하고,
> 모든 이미지에 **번호를 채번**해 번호 기준으로 사용·관리하는 시스템.
> 대시보드: `/admin/characters` · 번호 참조: `/c/HT-###`

---

## 1. 채번 규칙

| 항목 | 규칙 |
|---|---|
| 형식 | `HT-###` (3자리 0패딩, 999 초과 시 자릿수 자연 확장: HT-1000) |
| 발급 | DB `character_assets.num`(identity)이 자동 발급 — 수동 지정 금지 |
| 파생 | `code`는 `num`에서 생성되는 열(수정 불가) — **영구 식별자** |
| 결번 | 삭제된 번호는 재사용하지 않는다(참조 무결성) |
| 구간 | `HT-001`~`HT-105`: 시트 추출본(리포 정적 파일) · `HT-106`~: 대시보드 업로드 |

**번호 기반 사용법** — 파일 위치가 바뀌어도 주소가 유지된다:

```html
<img src="/c/HT-042" alt="오터 기본(행복)" />   <!-- 항상 최신 파일로 리다이렉트 -->
```

`/c/1`, `/c/ht-42` 같은 축약형도 정규화되어 동작한다. 비활성(숨김) 자산은 404.

## 2. 구성 요소

| 구성 | 위치 | 역할 |
|---|---|---|
| 추출 스크립트 | `scripts/build-character-assets.mjs` | 시트→컷 분리(Real-ESRGAN 4x 초해상→배경 투명화·슬리버 제거·2x 다운스케일), 레지스트리·콘택트시트 생성. **결정적**: 재실행해도 같은 번호=같은 컷 |
| 고화질 컷 | `public/characters/HT-###.png` | 투명 PNG, Real-ESRGAN 4x 초해상 → 2x 다운스케일(선명한 라인). 총 105개 ≈ 3.4MB |
| 레지스트리 | `src/data/characters.json` | 리포 자산 메타(번호·이름·그룹·크기·출처 좌표) — DB 폴백용 |
| DB | `character_assets` (0011·0012 마이그레이션) | 채번의 진실원본. RLS: 공개 읽기, 쓰기는 service_role만 |
| 스토리지 | Supabase `characters` 공개 버킷 | 대시보드 업로드 자산(`uploads/HT-###.<ext>`) |
| 번호 라우트 | `src/app/c/[code]/route.ts` | `/c/HT-###` → 이미지 리다이렉트(DB 우선, 레지스트리 폴백) |
| 헬퍼 | `src/lib/characters.ts` | 코드 정규화·조회·`characterUrl()` |
| 대시보드 | `/admin/characters` | 목록·검색·필터·복사·업로드(채번)·수정·숨김·삭제 |

## 4. 대시보드 사용법 (`/admin/characters`)

- **찾기**: 그룹 탭 + 검색(코드·이름·섹션·태그).
- **사용**: 카드의 `주소 복사`(`/c/HT-###`) · `<img> 복사` · `MD 복사`.
- **새 이미지 등록**: `+ 새 이미지 등록(채번)` → 파일(PNG 투명 권장, 최대 5MB)+이름 입력 →
  등록 즉시 번호가 발급되고 `/c/HT-###` 주소가 생긴다.
- **수정**: 이름·섹션·태그·메모 편집(번호·코드는 불변).
- **숨김/노출**: 숨기면 `/c/` 참조가 404 — 링크를 깨지 않고 회수 가능.
- **삭제**: 업로드 자산만 가능(스토리지 파일도 함께 정리, 번호는 결번). 시트 추출 자산은 숨김만.

## 5. 재추출(시트 수정 시)

```bash
node scripts/build-character-assets.mjs   # public/characters + characters.json 재생성
# 고품질 파이프라인: local/tools/realesrgan/ 에 Real-ESRGAN 실행 파일이 있으면
# 시트를 4x 초해상한 캐시(local/tools/sr/)를 만들어 사용, 없으면 란초스 폴백
# 검증: scripts/out/characters-contact.html 을 브라우저로 열어 라벨·크롭 확인
```

좌표는 스크립트 상단 `SHEETS` 밴드 정의로 관리한다. 새 시트/새 행 추가 시 **끝에만 추가**할 것 —
중간 삽입은 기존 번호를 밀어 채번 원칙(번호=영구 식별자)이 깨진다. 재생성 후에는 시드(0012)와
동일 규칙의 upsert SQL로 DB를 갱신한다.
